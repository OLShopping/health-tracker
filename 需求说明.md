# 个人健康追踪 Web 应用 — 需求说明

> 版本：v1.0  
> 整理日期：2026-03-27  
> 项目路径：`health-tracker/`

---

## 一、项目背景

本项目为个人使用的健康追踪工具，运行在家庭 NAS（飞牛 OS）或任意 Docker 环境中，通过手机或电脑浏览器访问，无需安装 App。

---

## 二、技术选型要求

| 层级 | 要求 |
|------|------|
| 前端 | 单文件 HTML（内嵌 CSS + JavaScript），无需打包构建 |
| 后端 | Python（Flask 框架） |
| 数据库 | SQLite（单文件，无需额外数据库服务） |
| 部署 | Docker + docker-compose，支持飞牛 NAS 和任意 Docker 环境 |
| 端口 | 5555（可通过环境变量 `PORT` 覆盖） |

---

## 三、功能模块需求

### 3.1 今日服药（服药打卡）

**核心需求：**
- 展示当日所有需要服用的药品及其时段配置
- **按时段分组展示**（早餐后 / 午餐后 / 晚餐后 / 其他），每个时段为一个区块，区块内列出该时段的所有药品
  > ⚠️ 重要：不能按药品名称分组；必须以时段为主维度组织界面

**时段区块显示内容：**
- 时段图标（🌅早 / ☀️午 / 🌙晚）+ 时段名称
- 该时段的服药状态摘要：✅ 已全部服用 / ⚠️ 部分已服 / 待服药

**每个药品行显示内容：**
- 药品图片缩略图（无图则显示💊图标）
- 药品名称（已服药变绿色高亮）
- 剂量（如：1片、1/2片）
- 库存数量（如：45片剩余）
- **剩余可服天数**（库存 ÷ 每日总剂量）
  - 颜色规则：> 3天显示灰色 / ≤ 3天显示橙色加⚠️警告 / 0天显示红色闪烁"⚠️ 已耗尽"
- 打卡状态图标：已服 ✅ / 未服 ○
- 整行可点击，点击后弹出精细剂量选择对话框

**打卡剂量选择：**
- 支持 1/4、1/3、1/2、整片等精细剂量
- 支持自定义备注

**数据联动（重要）：**
- 打卡成功后，页面库存数量和剩余天数必须**即时刷新**（不能有视觉滞后）
- 删除打卡记录后，库存必须即时恢复并刷新显示

---

### 3.2 库存管理

- 添加药品：名称、规格（单位）、当前库存数量、每日用药总量、备注
- 补充库存：输入补充数量，立即更新
- 进度条展示剩余天数（≤ 3天橙色、0天红色闪烁）
- 删除药品（含关联记录，不可撤销，需二次确认）
- 补充库存或删除后，今日服药页面同步刷新

---

### 3.3 药品照片

- 支持上传药品图片（PNG / JPG / GIF / WebP）
- 手机端可直接调用相机拍摄
- 图片在今日服药和库存管理页面作为缩略图展示
- 点击可放大预览

---

### 3.4 服药日历

- 月视图日历
- 颜色编码：
  - 🟢 绿色 = 当日全部服药完成
  - 🟡 黄色 = 当日部分漏服
  - 🔴 红色 = 当日有漏服
  - 灰色 = 无记录
- 点击某天，显示当日服药详情弹窗

---

### 3.5 血压心率记录

- 记录字段：收缩压、舒张压、心率、测量时间、备注
- 历史列表展示
- **图表趋势**：Chart.js 折线图，展示当日往前推一个月的数据
  - 三条折线：收缩压 / 舒张压 / 心率
  - X 轴为日期，Y 轴为数值

---

### 3.6 大便记录

- 一键记录（记录当前时间）
- 显示距上次排便的间隔时间（小时）
- 显示历史平均间隔
- 历史列表（可删除）

---

### 3.7 数据备份与恢复

- **JSON 完整备份**：导出全部数据为单个 JSON 文件
- **CSV 导出**：按模块分别导出（服药记录、库存、血压、大便记录）
- **JSON 数据恢复**：上传备份文件恢复数据，执行前弹出二次确认提示

## 四、界面与交互要求

### 响应式布局

| 设备 | 导航方式 | 布局特征 |
|------|----------|----------|
| 手机 | 底部 Tab 导航栏（6个Tab） | 单列卡片，大触摸区域 |
| 电脑 | 顶部横向导航栏 | 多列宽屏布局 |

### 其他交互规范

- 所有操作均有 Toast 提示反馈（成功 / 失败）
- 危险操作（删除、数据恢复）必须二次确认
- 数字输入框在手机上自动唤起数字键盘
- 药品照片上传在手机上可直接调用相机

## 五、数据存储

| 数据类型 | 存储位置 |
|----------|----------|
| 数据库 | `data/health.db`（SQLite，挂载到宿主机） |

```yaml
volumes:
  - /volume1/docker/health-tracker/data:/data
  - /volume1/docker/health-tracker/uploads:/app/static/uploads
```

## 七、部署方式

### 本地开发运行
```bash
cd health-tracker
pip install -r requirements.txt
DB_PATH=./data/health.db python app/main.py
# 访问 http://localhost:5555
```

### Docker 部署（推荐）
```bash
cd health-tracker
docker compose up -d
# 访问 http://宿主机IP:5555
```

### 飞牛 NAS 部署
1. 将 `health-tracker/` 文件夹上传到 NAS
2. 修改 `docker-compose.yml` 中 volumes 路径为 NAS 实际路径
3. 在 NAS 的 Docker 管理界面或 SSH 中执行：
   ```bash
   docker compose up -d
   ```
4. 局域网任意设备访问：`http://NAS的IP:5555`
