# 2026-03-29 日志

## 完成的功能

### 访问密码认证（首次会话遗留任务）
- 后端：新增 `@require_auth` 装饰器，`AUTH_PASSWORD` 环境变量控制，`/api/login`、`/api/auth-status` 接口
- 前端：登录遮罩（渐变背景 + 居中卡片），Token 存储 localStorage，所有 API 请求携带 `X-Auth-Token` 头部
- 401 时自动清除 Token 并刷新页面
- 下载（JSON/CSV）和恢复功能同步支持认证
- Docker compose 已添加 AUTH_PASSWORD 环境变量注释

## 遇到的问题
- Python 脚本对 main.py 做批量 `@require_auth` 注入时，因正则匹配问题导致多处函数定义被吞掉，文件损坏。手动逐个修复所有缺失的 `def` 函数定义行（15+ 处）。

## 技术细节
- 无密码时（`AUTH_PASSWORD=""`）：`api_auth_status` 返回 `require_auth: false`，登录遮罩不显示，所有 API 正常访问
- 有密码时：遮罩默认显示，输入正确密码后 Token 存入 localStorage，整个会话有效（刷新/重新打开需重新登录）
